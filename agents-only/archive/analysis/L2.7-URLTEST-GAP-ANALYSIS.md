## 一、Go 参考实现概要

### 1.1 核心架构

Go 的 URLTest 涉及 3 个组件：

| 组件 | 文件 | 职责 |
|------|------|------|
| `URLTestGroup` | `protocol/group/urltest.go` | 选择逻辑 + 健康检查循环 |
| `URLTestHistoryStorage` | `adapter/experimental.go` + `common/urltest/urltest.go` | 全局共享延迟历史存储 |
| `Clash API handlers` | `experimental/clashapi/proxies.go` + `api_meta_group.go` | 暴露历史到 API |

### 1.2 URLTestHistoryStorage

```go
// adapter/experimental.go
type URLTestHistory struct {
    Time  time.Time `json:"time"`
    Delay uint16    `json:"delay"`
}

type URLTestHistoryStorage interface {
    SetHook(hook chan<- struct{})
    LoadURLTestHistory(tag string) *URLTestHistory
    DeleteURLTestHistory(tag string)
    StoreURLTestHistory(tag string, history *URLTestHistory)
    Close() error
}
```

**关键特征**:
- **每 tag 仅 1 条记录** — `map[string]*URLTestHistory`，不是数组
- **全局共享** — URLTest 组和 Clash API server 引用同一实例
- **来源优先级**: context > ClashServer.HistoryStorage() > new instance
- **线程安全** — RWMutex 保护

### 1.3 URL 测试方法

```go
// common/urltest/urltest.go — URLTest()
func URLTest(ctx context.Context, link string, detour N.Dialer) (t uint16, err error) {
    // 1. 解析 URL, 提取 host:port
    // 2. 通过 outbound 建立 TCP 连接 (detour.DialContext)
    // 3. 创建 http.Client，Transport.DialContext 复用已建连接
    // 4. 发送 HTTP HEAD 请求 (不是 GET)
    // 5. TLS 支持 (http.Client 自动处理)
    // 6. 不跟随重定向 (http.ErrUseLastResponse)
    // 7. 返回 uint16 延迟毫秒数
}
```

| 参数 | Go 默认值 |
|------|----------|
| URL | `https://www.gstatic.com/generate_204` |
| Timeout | 15 秒 (`C.TCPTimeout`) |
| Method | HTTP HEAD |
| Concurrency | 10 goroutine (batch) |
| http:// URL | **拒绝**，替换为默认 HTTPS URL |

### 1.4 Tolerance / Sticky 选择逻辑

```go
// Select() — 每次连接时调用
func (g *URLTestGroup) Select(network string) (adapter.Outbound, bool) {
    var minDelay uint16
    var minOutbound adapter.Outbound

    // 1. 从当前已选的 outbound 开始 (sticky)
    if g.selectedOutboundTCP != nil {
        if history := g.history.LoadURLTestHistory(RealTag(g.selectedOutboundTCP)); history != nil {
            minOutbound = g.selectedOutboundTCP
            minDelay = history.Delay
        }
    }

    // 2. 仅当新候选者的延迟比当前低 tolerance 以上时才切换
    for _, detour := range g.outbounds {
        history := g.history.LoadURLTestHistory(RealTag(detour))
        if history == nil { continue }
        if minDelay == 0 || minDelay > history.Delay + g.tolerance {
            minDelay = history.Delay
            minOutbound = detour
        }
    }

    // 3. 无历史数据时兜底到第一个支持该网络的
    if minOutbound == nil { ... fallback ... }
    return minOutbound, true
}
```

**关键**: `minDelay > history.Delay + g.tolerance` — 新候选者必须比当前最优低 tolerance ms 才会切换，防止抖动。

### 1.5 健康检查触发机制

| 触发方式 | 说明 |
|----------|------|
| PostStart | 启动后立即触发一次 `CheckOutbounds(false)` |
| Touch() | 每次 DialContext/ListenPacket 时调用，创建 ticker + 更新 lastActive |
| loopCheck | ticker 周期（interval）检查: 超过 idleTimeout 则停止; 否则 CheckOutbounds |
| Force | Clash API `urlTestGroup.URLTest(ctx)` 强制测试 |

**跳过逻辑**: `!force && history != nil && time.Since(history.Time) < interval` → 跳过近期已测试的

### 1.6 失败处理

- **URL test 失败**: `DeleteURLTestHistory(tag)` — 删除历史（不是标记不健康）
- **DialContext 失败**: 同样 `DeleteURLTestHistory(tag)`
- **成功恢复**: 下次 URL test 成功时重新 `StoreURLTestHistory(tag, {...})`
- **选择更新后**: 中断现有连接 (`interruptGroup.Interrupt()`)

### 1.7 proxyInfo() 暴露

```go
func proxyInfo(server *Server, detour adapter.Outbound) *badjson.JSONObject {
    // history 查找使用 OutboundTag(detour)
    //   - 普通 proxy: detour.Tag()
    //   - group: group.Now() (当前选中成员)
    delayHistory := server.urlTestHistory.LoadURLTestHistory(adapter.OutboundTag(detour))
    if delayHistory != nil {
        info.Put("history", []*adapter.URLTestHistory{delayHistory})
    } else {
        info.Put("history", []*adapter.URLTestHistory{})
    }
    // 注意: Go 不设 alive/delay 顶级字段
    // GUI 从 history[last].delay 推导
}
```

## 二、Rust 当前实现状态

### 2.1 SelectorGroup 健康检查

| 方面 | 当前状态 |
|------|----------|
| `start_health_check()` | 仅 UrlTest 模式启动，weak ref 自动停止 |
| 循环方式 | 固定 interval sleep（无 idle timeout） |
| `run_health_checks()` | 对每个成员 spawn 独立 task（无并发限制） |
| `health_check_via()` | HTTPS: 仅 TCP connect; HTTP: 发送 HEAD + 读响应 |
| 默认 test URL | `http://cp.cloudflare.com`（与 Go 不同） |
| 默认 interval | 10 min（Go: 3 min） |
| 默认 timeout | 5 sec（Go: 15 sec） |

### 2.2 ProxyHealth

```rust
pub struct ProxyHealth {
    is_alive: Arc<parking_lot::RwLock<bool>>,
    last_rtt_ms: AtomicU64,
    consecutive_fails: AtomicUsize,
    last_check: Mutex<Option<Instant>>,       // 单调时钟，不可序列化
    active_connections: AtomicUsize,
    permanent_failure: AtomicBool,
    last_error: Mutex<Option<String>>,
}
```

**缺失**:
- 无 `SystemTime` 时间戳（无法生成 ISO 8601 给 API）
- 无共享的历史存储（`ProxyHealth` 是 `SelectorGroup` 内部 per-member 结构）
- `record_failure()` 递增 `consecutive_fails`（Go 是删除历史记录）

### 2.3 select_by_latency()

```rust
fn select_by_latency(&self) -> Option<&ProxyMember> {
    let healthy = self.members.iter().filter(|m| m.health.is_healthy());
    let best = healthy.iter().min_by_key(|m| m.health.get_rtt_ms()).copied();
    let _ = self.tolerance_ms; // 标记为故意未使用
    best
}
```

**`tolerance_ms` 完全未使用**。总是选绝对最低延迟，无 sticky 防抖。

### 2.4 Clash API 状态

`ApiState` 中**无任何历史存储字段**。

### 2.5 API handlers

| handler | 当前行为 | Go 行为差异 |
|---------|----------|------------|
| `get_proxies` / `get_proxy` | `history: vec![]`, `alive: Some(true)` | Go 从 `urlTestHistory` 填充 history |
| `get_proxy_delay` | 返回 `{"delay":N}` 但不存储 | Go 存储到 `urlTestHistory` |
| `get_meta_group_delay` | 返回 `{tag:delay}` 但不存储 | Go 存储到 `urlTestHistory` |

| # | 缺口 | 严重度 | 说明 |
|---|------|--------|------|
| G1 | **无 URLTestHistoryStorage** | 🔴 Critical | 无全局共享延迟历史。Go 有 `HistoryStorage` map keyed by tag |

## 五、关键设计决策点

### 5.1 HistoryStorage 放在哪个 crate?

**选项 A**: `sb-core` — 与 SelectorGroup 同级
- 优点: SelectorGroup 可直接引用
- 缺点: sb-api 也需要引用，增加耦合

**选项 B**: `sb-types` — 作为 port trait
- 优点: 契约层，sb-core 和 sb-api 都可引用
- 缺点: sb-types 不应有运行时逻辑

**选项 C**: `sb-core` 定义 trait，`sb-core` 实现，通过 ContextRegistry 传递（与 CacheFile 模式一致）
- 优点: 已有先例模式
- 推荐: ✅

### 5.2 ProxyHealth 是否需要改动?

**建议**: 保持 `ProxyHealth` 不变。新增独立的 `URLTestHistoryStorage` 作为全局存储。
理由: ProxyHealth 是 per-member 内部状态，HistoryStorage 是全局共享 API 层状态，职责不同。

### 5.3 history 数组长度

Go 只存 1 条记录，API 返回 `[{time, delay}]` 或 `[]`。GUI 只读 `history[last]`。
**建议**: 与 Go 对齐，每 tag 仅 1 条。

### 5.4 OutboundTag 逻辑

Go: `OutboundTag(group) = group.Now()` — group 的 history 查找键是当前选中成员的 tag。
这意味着 URLTest group 显示的是其最佳成员的延迟。
**建议**: 使用 `OutboundGroup::now()` 作为 lookup key（L2.6 已有 `as_group().now()`）。
