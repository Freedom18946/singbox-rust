## 一、Go 参考实现概要

### 1.1 三层架构

Go 连接跟踪涉及三个独立层：

| 层 | 组件 | 文件 | 职责 |
|----|------|------|------|
| L1 | `conntrack` (OS 级) | `common/conntrack/` | 全局连接列表，OOM kill switch（build-tag 保护，可选） |
| L2 | `trafficontrol.Manager` | `experimental/clashapi/trafficontrol/` | Clash API 连接追踪 + 流量统计 |
| L3 | `route.ConnectionManager` | `route/` | 双向数据拷贝 / 连接生命周期管理 |

数据流:
```
Inbound → Router.routeConnection()
            │
            ├─ conntrack.KillerCheck()         // L1: OOM 保护（可选）
            │
            ├─ matchRule() → selectedOutbound
            │
            ├─ for _, tracker := range r.trackers:
            │     conn = tracker.RoutedConnection(conn, metadata, rule, outbound)
            │                                    // L2: 包装连接加计数器
            │
            └─ connectionManager.NewConnection() // L3: 建连 + 双向拷贝
```

### 1.2 核心接口

#### ConnectionTracker（连接包装接口）

```go
// adapter/router.go
type ConnectionTracker interface {
    RoutedConnection(ctx, conn, metadata, matchedRule, matchOutbound) net.Conn
    RoutedPacketConnection(ctx, conn, metadata, matchedRule, matchOutbound) N.PacketConn
}
```

Router 遍历所有注册的 tracker，每个 tracker 用计数 wrapper 包装连接：
```go
for _, tracker := range r.trackers {
    conn = tracker.RoutedConnection(ctx, conn, metadata, selectedRule, selectedOutbound)
}
```

ClashServer 和 V2RayStatsService 都实现此接口。

#### ConnectionManager（连接生命周期）

```go
// adapter/connections.go
type ConnectionManager interface {
    Lifecycle
    NewConnection(ctx, dialer, conn, metadata, onClose)
    NewPacketConnection(ctx, dialer, conn, metadata, onClose)
}
```

### 1.3 TrackerMetadata 结构

```go
// trafficontrol/tracker.go
type TrackerMetadata struct {
    ID           uuid.UUID
    Metadata     adapter.InboundContext
    CreatedAt    time.Time
    ClosedAt     time.Time
    Upload       *atomic.Int64
    Download     *atomic.Int64
    Chain        []string
    Rule         adapter.Rule
    Outbound     string
    OutboundType string
}
```

关键特征:
- **Upload/Download**: 原子计数器，与 counter-wrapped conn 共享引用
- **Chain**: 从 matched outbound 走到 leaf（via `OutboundGroup.Now()`），然后**反转**
- **ClosedAt**: 连接关闭时由 `Leave()` 设置

### 1.4 trafficontrol.Manager 结构

```go
// trafficontrol/manager.go
type Manager struct {
    uploadTotal   atomic.Int64       // 全局累计上传
    downloadTotal atomic.Int64       // 全局累计下载
    connections   compatible.Map[uuid.UUID, Tracker]  // 活跃连接 (sync.Map)
    closedConnections list.List[TrackerMetadata]       // 已关闭连接，上限 1000 (FIFO)
    memory        uint64
}
```

核心方法:
- `Join(tracker)`: 注册活跃连接
- `Leave(tracker)`: 移入 closedConnections（cap=1000），设置 ClosedAt
- `Snapshot()`: 返回 `{downloadTotal, uploadTotal, connections[], memory}` — **过滤 DNS 类型连接**
- `PushUploaded(n)` / `PushDownloaded(n)`: 累加全局计数
- `Total() → (up, down)`: 读取全局计数

### 1.5 per-connection 字节计数

```go
// tracker.go — NewTCPTracker
ExtendedConn: bufio.NewCounterConn(conn, []N.CountFunc{func(n int64) {
    upload.Add(n)           // per-connection
    manager.PushUploaded(n) // 全局
}}, []N.CountFunc{func(n int64) {
    download.Add(n)           // per-connection
    manager.PushDownloaded(n) // 全局
}})
```

每次 read/write 触发**两个原子加**：per-connection + 全局。

方向语义:
- **Upload** = 从 inbound 读取的字节（客户端 → 代理）
- **Download** = 写入 inbound 的字节（代理 → 客户端）

### 1.6 Chain 构建逻辑

```go
// tracker.go
next = matchOutbound.Tag()   // 或 outboundManager.Default().Tag()
for {
    detour, loaded := outboundManager.Outbound(next)
    if !loaded { break }
    chain = append(chain, next)
    group, isGroup := detour.(adapter.OutboundGroup)
    if !isGroup { break }
    next = group.Now()
}
common.Reverse(chain)  // [matched→...→leaf] 变为 [leaf→...→matched]
```

### 1.7 Snapshot JSON Schema

**顶层:**
```json
{
    "downloadTotal": 123456,     // int64, 全局累计
    "uploadTotal": 789012,       // int64, 全局累计
    "connections": [...],        // TrackerMetadata 数组
    "memory": 52428800           // uint64, Go runtime 内存
}
```

**每个 connection:**
```json
{
    "id": "uuid-v4-string",
    "metadata": {
        "network": "tcp",
        "type": "mixed/mixed-in",
        "sourceIP": "192.168.1.100",
        "sourcePort": "12345",         // 字符串！
        "destinationIP": "1.2.3.4",
        "destinationPort": "443",      // 字符串！
        "host": "example.com",
        "dnsMode": "normal",           // 始终 "normal"
        "processPath": "/usr/bin/curl"
    },
    "upload": 1024,
    "download": 4096,
    "start": "2024-01-01T00:00:00Z",   // RFC3339
    "chains": ["proxy", "selector"],    // 反转后的 chain
    "rule": "domain_suffix(example.com) => route(proxy)",
    "rulePayload": ""                   // 始终空字符串
}
```

### 1.8 /traffic WebSocket

发送**每秒 delta**（不是绝对值）:
```go
tick := time.NewTicker(time.Second)
uploadTotal, downloadTotal := trafficManager.Total()
for range tick.C {
    newUp, newDown := trafficManager.Total()
    write(Traffic{Up: newUp - uploadTotal, Down: newDown - downloadTotal})
    uploadTotal, downloadTotal = newUp, newDown
}
```

格式: `{"up": <bytes_per_second>, "down": <bytes_per_second>}`

### 1.9 /connections WebSocket

每秒推送完整 Snapshot：
```go
tick := time.NewTicker(1000ms)  // 可通过 ?interval= 参数调整
for range tick.C {
    snapshot := trafficManager.Snapshot()
    write(snapshot)
}
```

HTTP 请求（无 WS Upgrade）直接返回单次 Snapshot。

### 1.10 连接关闭

**DELETE /connections/:id**: 遍历 snapshot 找到匹配 UUID，调用 `tracker.Close()`（先 `Leave()` 再关 socket），**始终 204**。

**DELETE /connections**: 关闭所有连接 + `router.ResetNetwork()`。

### 1.11 Wiring

```go
// box.go
clashServer := experimental.NewClashServer(...)
router.AppendTracker(clashServer)  // 注册为 ConnectionTracker
```

### 2.2 WebSocket 行为

GUI 打开 3 个长活 WebSocket: `/memory`, `/traffic`, `/connections`。
- `/connections` WebSocket 推送完整 `CoreApiConnectionsData` 快照
- GUI 通过 diff 连续快照检测断开的连接（前一个 snapshot 有、当前没有 → 移入 "Closed" 标签页）
- GUI 在客户端计算 per-connection 速率: `speed = current.upload - previous.upload`

### 2.3 UI 渲染 (12 列)

| 列 | 数据来源 | 显示逻辑 |
|----|---------|---------|
| Type | `metadata.type` | `{type}({network})` |
| Process Path | `metadata.processPath` | 原样 |
| Host | `metadata.host` | 空时 fallback 到 `metadata.destinationIP` |
| Source | `metadata.sourceIP` | `{sourceIP}:{sourcePort}` |
| Destination IP | `metadata.destinationIP` | `{destinationIP}:{destinationPort}` |
| Rule | `rule` | `{rule}::{rulePayload}` |
| Chains | `chains` | 反转后用 ` :: ` 连接 |
| UL Speed | computed | `record.upload - record.up` (delta) |
| DL Speed | computed | `record.download - record.down` (delta) |
| Upload | `upload` | 总字节数 |
| Download | `download` | 总字节数 |
| Time | `start` | `formatRelativeTime()` — 必须是可解析的日期 |

### 2.4 GUI 关键行为

1. **代理切换时自动关闭连接**: `GET /connections` → 过滤 `chains` 含 group name 的 → `DELETE /connections/{id}` 逐个关闭
2. **模式切换时全部关闭**: 调用 `DELETE /connections/{id}` 逐个（不是 `DELETE /connections`）
3. **`start` 格式**: 必须是 ISO 8601 或数字时间戳（当前 Rust 返回 Unix ms 字符串 `"1705312200000"` 在某些 JS 引擎上 `new Date("1705312200000")` 返回 Invalid Date）

## 五、设计决策点

### 5.1 架构选择：ConnectionTracker trait vs 直接注入

**选项 A**: Go 模式 — 定义 `ConnectionTracker` trait，Router 持有 `Vec<Arc<dyn ConnectionTracker>>`
- 优点: 与 Go 架构一致，支持多个 tracker (Clash + V2Ray)
- 缺点: 需要改 Router，影响面大

**选项 B**: 直接注入 — 在现有连接处理路径中直接调用 API `ConnectionManager`
- 优点: 改动最小，利用现有 `sb-api/managers.rs::ConnectionManager`
- 缺点: Clash API 特定逻辑渗入 core 层

**选项 C**: 共享 `ConnTracker` (sb-common) — 作为中间层
- 优点: sb-common 已有 `ConnTracker` + `ConnMetadata` + DashMap
- 缺点: 需要加流量计数，API 层需要适配

**推荐**: **选项 A 简化版** — 在 sb-core 定义 trait，但 L2.8 仅实现 Clash API tracker（不做 V2Ray），通过 Context 注入。

### 5.2 字节计数方案

**选项 A**: Counter wrapper (Go 模式) — 包装 `TcpStream` / `AsyncRead+AsyncWrite`
- 优点: 精确计数，与 Go 完全一致
- 缺点: 需要在 tokio I/O path 加 wrapper，影响性能

**选项 B**: 在双向拷贝层计数 — 修改 `RouteConnectionManager::copy_with_recording()`
- 优点: 改动集中，性能影响小
- 缺点: 只能在拷贝层计数，不含 TLS 开销

**推荐**: **选项 B** — 在已有的 `copy_with_recording()` / `new_connection()` 中加入原子计数即可。

### 5.3 Close 信号传播

**选项 A**: `CancellationToken` per connection
- 优点: tokio 原生，clean cancellation
- 缺点: 需要在每个连接创建时传入 token

**选项 B**: 直接 `abort()` JoinHandle
- 优点: 简单
- 缺点: 不优雅，可能泄露资源

**选项 C**: 在 ConnectionManager 中存储 `oneshot::Sender<()>`
- 优点: 与现有架构一致
- 缺点: 只能关闭一次

**推荐**: **选项 A** — 每个连接持有 `CancellationToken`，close 时 cancel。

### 5.4 哪些现有代码可以复用

| 代码 | 复用方式 |
|------|---------|
| `sb-api/managers.rs::Connection` | 修改: 加 `CancellationToken`, 保留 `Arc<AtomicU64>` counters |
| `sb-api/managers.rs::ConnectionManager` | 修改: 加全局 atomic counters, closed list |
| `sb-api/types.rs::Connection` | 保持: API 输出类型已正确 |
| `sb-api/clash/websocket.rs` | 扩展: 加 connections WS handler |
| `sb-core/router/conn.rs` | 修改: 注册连接 + 字节计数 |

### 5.5 不做什么（延后到 L2.8+）

- conntrack OOM kill switch (L1 级，可选)
- V2Ray StatsService per-inbound/per-outbound 统计
- 进程检测 (ProcessMatcher 接入)
- closed connections 历史列表 (1000 上限)
- DNS 连接过滤
- idle timeout / freshness skip
