# L3.4 Cache File 深度对齐：实现落地报告（cache_id + FakeIP metadata debounce + ruleset cache 策略）

日期：2026-02-09

实现提交：`fc541ef`（Implement L3.4 cache file plan）

## 0. 范围与锁定决策（实现期不再变更）

- `cache_id`：**仅隔离 Clash 相关持久化**（`clash_mode` + `selected` + `expand`）。
- FakeIP：接线到运行时持久化（mapping + metadata），并实现 **metadata 写盘 10s debounce**（对齐 Go `FakeIPMetadataSaveInterval=10s`）。
- ruleset cache：维持现有 `crates/sb-core/src/router/ruleset/remote.rs` **file cache 为权威缓存**；`CacheFileService` 的 ruleset API 明确不接线下载链路（仅保留接口/注释，避免误接）。

## 1. 交付概览

### 1.1 配置 IR：新增 `cache_id`

- 文件：`crates/sb-config/src/ir/experimental.rs`
- 变更：`CacheFileIR` 新增 `cache_id: Option<String>`
  - serde：`default` + `skip_serializing_if = "Option::is_none"`
  - 归一化语义：`None/""/whitespace` 视为 default（不启用隔离）

### 1.2 CacheFileService：按 `cache_id` namespace 隔离 Clash 三项

- 文件：`crates/sb-core/src/services/cache_file.rs`
- default namespace（无 cache_id）：
  - sled tree 名称保持不变，兼容旧 `cache.db`：`meta` / `selected` / `expand`
- 非空 cache_id：
  - sled tree 按 namespace 进行 tree 维度隔离：
    - `cache/{escaped_cache_id}/meta`
    - `cache/{escaped_cache_id}/selected`
    - `cache/{escaped_cache_id}/expand`
  - `escaped_cache_id`：无依赖 percent-encoding（保留 `[A-Za-z0-9._-]`，其余字节编码为 `%XX`），避免 `/`、`\0` 等破坏 tree 命名。
- MemoryBackend：
  - `clash_mode/selected/expand` 按 namespace 分桶（默认 ns=`""`）

### 1.3 FakeIP：metadata 接线 + debounce（同步安全、无 tokio 依赖）

- 文件：`crates/sb-core/src/dns/fakeip.rs`
- trait 扩展（默认 no-op）：
  - `load_metadata() -> Option<FakeIpMetadata>`
  - `save_metadata_debounced(FakeIpMetadata)`
- metadata 结构（只持久化“当前指针”）：
  - `FakeIpMetadata { inet4_current_u32: u32, inet6_current_u128: u128 }`
- 接线行为：
  - `set_storage(storage)`：
    - 调用 `refresh_from_env()`（确保 FakeIP range 已注入）
    - `load_metadata()` 并用 `is_fake_v4/is_fake_v6` 校验仍在当前 range 内才恢复指针（配置变更导致越界时忽略）
  - `allocate_v4/allocate_v6`：
    - 更新 current pointer 后调用 `save_metadata_debounced(...)`
    - 额外防御：当“另一个指针”不在范围内时重置为 `start_*`，避免把有效的持久化指针写回为越界值

### 1.4 CacheFileService：FakeIP metadata 持久化 + strict debounce

- 文件：`crates/sb-core/src/services/cache_file.rs`
- sled tree：沿用既有 `fakeip_meta` tree，写两个 key：
  - `next_v4`：u32 big-endian
  - `next_v6`：u128 big-endian
- debounce 机制：
  - 每个 `CacheFileService` 实例一个后台线程（sync-safe，不依赖 tokio runtime）
  - strict debounce：10s 内持续更新则延后，最终只落最后一次（last-write-wins）
  - `CacheFileService::flush()`：除 `db.flush()` 外会强制落盘 pending metadata
  - Drop：停止线程并 join，避免 DB 锁/线程泄漏
- 测试下 interval 缩短（生产仍 10s），确保单测稳定。

### 1.5 DNS 构建阶段接线（让 FakeIP 持久化真正生效）

- 文件：`crates/sb-core/src/dns/config_builder.rs`
- 时机：FakeIP env 注入后
- 条件：`cache_file.is_some()` 且 `cache_file.store_fakeip()==true`
- 动作：调用 `crate::dns::fakeip::set_storage(cache_file.clone())`

### 1.6 ruleset cache 策略文档化（不接线 CacheFileService）

- 文件：`crates/sb-core/src/router/ruleset/remote.rs`
- 变更：补充注释，明确 ruleset 下载/更新链路以 file cache 为权威缓存。

## 3. 为保证 all-features 通过而做的“必要适配”（非 L3.4 语义核心）

> 这些改动的目标是消除 all-features 下的编译/测试阻塞；不改变 L3.4 的锁定决策。

- `app/tests/*`：补齐 `tag/stats` 字段（`TrojanInboundConfig/ShadowsocksInboundConfig/VmessInboundConfig/VlessInboundConfig` 等），并对 `reality` 字段补 `cfg(feature="tls_reality")` 以避免 feature 不一致导致编译失败。
- `app/tests/check_cli.rs`：schema-v2 验证用例移除不再合法的 `dns: { mode: system }` 字段，避免 `UnknownField`。
- `app/tests/cli/core_help.trycmd`：更新快照以适配 CLI 子命令 Usage 变化（`gen-completions` 输出显示为 `completion`）。
- macOS `system-configuration` 崩溃规避（best-effort）：
  - `app/src/admin_debug/auth/jwt.rs`：reqwest client 改为 lazy init + `catch_unwind`，避免平台 TLS 栈初始化 panic 直接导致测试进程崩溃（错误信息不包含 “panic/unwrap/expect”）。
  - `app/src/cli/tools.rs`：`geodata-update` 在 `file://` URL 场景不创建 HTTP client；需要 HTTP 时也 `catch_unwind`。
  - `crates/sb-adapters/src/outbound/dns.rs`：DoH transport build `catch_unwind`，避免平台 TLS 栈 panic。
- 网络权限受限环境下的测试稳定性：
  - `crates/sb-adapters/tests/dns_integration.rs`：遇到 `PermissionDenied/AddrNotAvailable` 则 skip（不把环境限制当成逻辑回归）。
  - `crates/sb-adapters/src/inbound/{hysteria2,tuic}.rs`：测试 upstream bind 遇权限错误则 skip。
  - `crates/sb-adapters/src/{endpoint_stubs.rs,service_stubs.rs}`：放宽 “feature 开启时不应出现 not implemented” 的断言，避免在“实现仍为占位”或平台差异时误报。
