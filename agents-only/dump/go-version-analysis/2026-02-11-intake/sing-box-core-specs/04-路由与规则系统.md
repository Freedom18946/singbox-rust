# 04 路由与规则系统（Route / Rules / Rule-Set）

路由系统像“心智”：它把一团混沌的流量，分门别类地导向不同的命运。sing-box 的路由之所以强，是因为它不仅看 IP/端口，还尝试看“语义”（域名、协议、进程、网络状态）。

## 1) Route 模块目标

- MUST：在连接建立前/中做出可解释的路由决策（选择 Outbound、执行 DNS resolve、拒绝/阻断等）。
- MUST：支持规则优先级（按 rules 数组顺序匹配）与“默认规则逻辑”。
- SHOULD：规则引擎应可扩展（新增 rule item / action 不破坏旧配置）。

## 2) Route Rule：匹配字段（Match Items）

### 2.1 基础匹配域

- MUST：支持按域名维度匹配：`domain`、`domain_suffix`、`domain_keyword`、`domain_regex`。
- MUST：支持按 IP/网段匹配：`ip_cidr`、`ip_is_private`，以及源地址版本。
- MUST：支持按端口匹配：`port`、`port_range`，以及源端口版本。

### 2.2 语义增强与环境感知

- MUST：支持 `protocol`（嗅探到的协议）与 `client`（嗅探到的客户端类型）匹配。
- MUST：支持进程/应用维度匹配（桌面/移动端场景）：  
  `process_name`、`process_path`、`process_path_regex`、`package_name`、`user`、`user_id`。
- SHOULD：支持网络环境维度匹配：  
  `network_type`、`network_is_expensive`、`network_is_constrained`、接口地址相关字段。  
- SHOULD：支持 Wi‑Fi 维度匹配：`wifi_ssid`、`wifi_bssid`（需要平台支持，详见 Wi‑Fi State）。

### 2.3 network：TCP/UDP/ICMP

- MUST：支持按网络层类型匹配：`tcp`、`udp`，并支持在新版本中引入的 `icmp`（用于 ping 等）。
- MUST：当流量来自 TUN/WireGuard/Tailscale 入站时，应能够把 ICMP 路由到 Direct/WireGuard/Tailscale 等出站（文档明确说明这一链路）。

### 2.4 逻辑规则（Logical Rule）

- MUST：支持 `type: logical` 规则，至少实现 AND/OR 等组合逻辑与嵌套规则数组。

## 3) Rule Action：动作（Action）

- MUST：支持“路由到指定 outbound”的动作（route）。
- MUST：支持 `resolve` 动作：在路由阶段把目的域名解析为 IP，可指定 DNS server tag 与解析策略（prefer_ipv4 / prefer_ipv6 / ipv4_only / ipv6_only），并可控制缓存/TTL/EDNS Client Subnet 等行为。

> 备注：历史上部分“特殊 outbound”（如 block/dns）在 Migration 中被归并为 Rule Action 体系；实现需遵循当前推荐方式。

## 4) Protocol Sniff：协议/客户端嗅探

- MUST：支持协议嗅探能力，并至少覆盖文档列出的协议：`tls`、`http`、`quic`、`stun`、`bittorrent`、`dns`、`ssh`、`rdp`、`ntp`。
- MUST：支持 QUIC Client 类型识别：`chromium`、`safari`、`firefox`、`quic-go`（用于更细粒度路由策略）。
- SHOULD：提供 sniff 的超时、缓冲大小与开关（具体字段在各 Inbound 或 Route 配置中体现），并在失败时回退到“不嗅探”的安全策略。

## 5) Rule-Set：规则集体系

### 5.1 Source Format（源格式）

- MUST：支持 source format 的规则集描述与加载。
- MUST：提供编译为 `.srs`（binary rule-set）的工具链（详见 01）。

### 5.2 Headless Rule（无头规则）

- SHOULD：支持“规则头信息缺失”的导入方式（官方给出 headless rule 的存在与用法），用于简化外部规则集接入。

### 5.3 AdGuard DNS Filter

- SHOULD：支持将 AdGuard DNS Filter 规则转换为 sing-box rule-set（convert），用于 DNS/拦截类策略统一管理。

## 6) 验收清单

- Route Rule 的字段匹配逻辑符合“默认匹配逻辑”描述，不出现意外短路。
- `resolve` 动作对 DNS 路由有清晰覆盖关系（指定 server 优先于 DNS 路由选择）。
- sniff 能识别至少 9 种协议与 QUIC 客户端类型；失败时不阻塞连接。
- rule-set 编译/加载/引用（by tag）完整闭环。
