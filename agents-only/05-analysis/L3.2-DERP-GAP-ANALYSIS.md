# L3.2 DERP 配置对齐（PX-014）前置信息收集与差距分析

> 日期: 2026-02-09  
> 目标: 在不改 DERP 协议实现本体的前提下，对齐 Go sing-box 的 **DERP 配置 schema + 关键运行时语义**（verify_client_url / mesh_with / ListenOptions / STUN / bootstrap-dns）。

---

## 1. 参考基线（Go）

代码:
- `go_fork_source/sing-box-1.12.14/service/derp/service.go`
- `go_fork_source/sing-box-1.12.14/option/tailscale.go`（DERPServiceOptions / DERPVerifyClientURLOptions / DERPMeshOptions / DERPSTUNListenOptions）

文档:
- `go_fork_source/sing-box-1.12.14/docs/configuration/service/derp.md`
- `go_fork_source/sing-box-1.12.14/docs/configuration/shared/dial.md`（Dial Fields）
- `go_fork_source/sing-box-1.12.14/docs/configuration/shared/listen.md`（Listen Fields）

关键语义（摘自 Go 实现）:
- `verify_client_url`:
  - 支持 `["https://x/verify"]` 或 `[{ "url": "...", ...dial fields }]` 两种格式（Listable + string/object 兼容）。
  - **每条 URL** 创建独立 dialer（Dial Fields）+ 独立 `http.Client`，通过 `Transport.DialContext` 使用该 dialer。
  - `TLSClientConfig` 根证书与时间函数来自 context（RootPool + NTP TimeFunc）。
- `mesh_with`:
  - 支持结构体数组（`server/server_port/host/tls/...dial fields`），每个 peer 有独立 dialer/TLS 选项。
  - mesh client 通过 derphttp client `RunWatchConnectionLoop` 订阅 `WatchConns`，并将 forwarder 注册进 server。
- `bootstrap-dns`:
  - 通过 context 注入的 `DNSRouter.Lookup(ctx, domain, ...)`（不是全局 resolver）。
- STUN:
  - 仅当 `stun` 配置存在且 `enabled=true` 时启用。
  - STUN 默认 listen=`::`，listen_port=`3478`（当启用且字段未填写时）。
- ListenOptions:
  - DERP TCP listener 使用 Listen Fields。
  - STUN UDP listener 使用 Listen Fields（来自 `stun.listen_options`）。

## 2. Rust 现状（实现与 IR）

实现:
- `crates/sb-core/src/services/derp/server.rs`

IR:
- `crates/sb-config/src/ir/mod.rs`：`ServiceIR` DERP 字段
  - `verify_client_endpoint: Option<Vec<String>>`
  - `verify_client_url: Option<Vec<String>>`（只有 string 列表）
  - `mesh_with: Option<Vec<String>>`（只有 string 列表）
  - `stun: Option<DerpStunOptionsIR>`（结构体，未做 “number/bool” 兼容）
  - 共享 Listen 字段存在于 `ServiceIR`（`bind_interface/routing_mark/reuse_addr/netns/...`），但 DERP 实现未使用。

运行时要点（从 Rust 代码直接可见）:
- `verify_client_url`:
  - 使用 `reqwest::Client::builder()`，没有 dialer 注入，也没有按 URL 级别应用 dial fields。
- `mesh_with`:
  - 仅支持 `Vec<String>` peers（`host:port` 字符串）。
  - peer 连接路径固定为 `TcpStream::connect(peer)` + TLS（root=webpki_roots + “把本机证书加入 roots”）+ HTTP Upgrade `GET /derp`。
  - 不支持 per-peer TLS options / host override / dialer options。
- `bootstrap-dns`:
  - 使用 `crate::dns::global::get()`（全局 resolver），`ServiceContext` 目前未接线使用。
- STUN:
  - 若 `stun` 字段缺失，当前实现默认为 `(enabled=true, addr=DERP listen_addr)`（与 Go 不一致）。
  - STUN listen options（bind_interface/routing_mark/netns/reuse_addr）在 `DerpStunOptionsIR` 中存在，但实现未使用。
- ListenOptions:
  - DERP TCP 监听使用 `StdTcpListener::bind(listen_addr)`。
  - 未 honor `bind_interface/routing_mark/reuse_addr/netns` 等字段。

## 3. 差距清单（Go 对齐目标）

### 3.1 配置 schema 差距（必须先补齐）

1. `verify_client_url`:
   - Go: `Listable[string | {url + dial fields}]`
   - Rust: `Vec<String>`

2. `mesh_with`:
   - Go: `Listable[{server/server_port/host/tls + dial fields}]`
   - Rust: `Vec<String>`

3. `stun`:
   - Go: `bool | number | object`（`DERPSTUNListenOptions` 自定义 marshal/unmarshal）
   - Rust: 仅 object（`DerpStunOptionsIR`），且默认行为与 Go 不一致。

4. `verify_client_endpoint` 语义:
   - Go: “Tailscale endpoint tag”（PostStart 从 EndpointManager 取 LocalClient）
   - Rust: 当前实现按 “socket path / http endpoint” 直接请求 whois（语义不一致）。

### 3.2 运行时语义差距（按 L3.2 子任务推进）

1. verify client:
   - 需要 “每条 URL 独立 http client + DialContext”。
   - 需要接入 dial fields（至少 bind_interface / routing_mark / netns / detour 等能影响出站路径的项）。

2. mesh peer:
   - 需要 per-peer dialer 与 per-peer TLS options（含 host override / SNI）。
   - 需要按 Go 的 server/server_port/host 拼装 URL 与 Host/SNI 语义。

3. `/bootstrap-dns`:
   - 需要走可注入 resolver/router（Go 是 DNSRouter），避免全局依赖。
   - Rust `ServiceContext` 已有 `dns_resolver` 字段，但 DERP builder/handler 未使用它。

4. ListenOptions / STUN:
   - DERP TCP 与 STUN UDP 都需要 honor listen fields（bind_interface / routing_mark / reuse_addr / netns 等）。
   - STUN 默认值需按 Go：仅在配置启用时启动；启用时默认 listen=`::`，port=`3478`。

## 4. 关键落点与最小改动策略（建议）

### 4.1 IR 扩展（建议设计）

目标: 先做到 “能解析 Go 配置且保留信息”，再逐步把信息真正用起来。

建议新增（位置: `crates/sb-config/src/ir/mod.rs`）:
- `DerpVerifyClientUrlIR`:
  - 字段: `url: String`
  - 字段: `dial: Option<DerpDialOptionsIR>`（或 `serde_json::Value` + 后续解释）
  - 反序列化兼容: string -> `{ url: string }`（serde untagged / custom deserialize）。
- `DerpMeshPeerIR`:
  - 字段: `server: String`, `server_port: Option<u16>`, `host: Option<String>`
  - 字段: `tls: Option<DerpOutboundTlsIR>`（需要定义 outbound TLS 选项容器，至少覆盖 mesh 所需字段）
  - 字段: `dial: Option<DerpDialOptionsIR>`
  - 反序列化兼容: string -> `{ server: string }`（保留旧配置最小兼容）。
- `ServiceIR`:
  - `verify_client_url: Option<Vec<DerpVerifyClientUrlIR>>`
  - `mesh_with: Option<Vec<DerpMeshPeerIR>>`
  - `stun: Option<DerpStunOptionsIR>` 继续保留，但需要支持 `bool/number/object` 的反序列化以对齐 Go（或新增一个 untagged wrapper）。

Dial/TLS 选项范围建议（分两阶段）:
- Phase A（最小可用）: 仅实现 Rust 已有且运行时可真正应用的字段（`bind_interface/routing_mark/reuse_addr/netns/tcp_fast_open/tcp_multi_path/udp_fragment/detour`）。
- Phase B（完整 schema 对齐）: 补齐 Go dial.md 的字段（inet4/inet6 bind addr, connect_timeout, domain_resolver, network_strategy/type...）。

### 4.2 Runtime 接线（建议落点）

位置: `crates/sb-core/src/services/derp/server.rs`
- `DerpService::from_ir(ir, ctx)` 不再忽略 `ctx`:
  - 将 `ctx.dns_resolver` 注入到 HTTP handler state，用于 `/bootstrap-dns`。
  - 预留 verify/mesh 的 “dialer factory” 注入口（未来可从 context 注入，或由 derp 自己构造 direct connector）。
- `/bootstrap-dns`:
  - 从 “global resolver” 切换为 “state 持有的 resolver”。
  - 增加单测：不安装 global 的情况下，注入 fake resolver 仍可工作（对齐 L3-WORKPACKAGES 的验收点）。
- ListenOptions / STUN:
  - 参考 `crates/sb-core/src/services/ssmapi/server.rs` 的 socket 选项设置方式（reuse_addr / routing_mark / bind_interface）。
  - DERP TCP / STUN UDP 的 socket 预绑定阶段就应用选项，失败要同步冒泡（保持当前“预绑定”策略）。

### 4.3 verify_client_endpoint 语义对齐（需要一个小决策）

Go 的语义是 “endpoint tag -> tailscale LocalClient”。Rust 当前 `ServiceContext` 没有 EndpointManager 注入。

建议选项:
1. **严格对齐 Go（推荐）**:
   - 扩展 `ServiceContext`：注入 `endpoint_manager`（或注入 “tag -> LocalClient” 适配后的 trait）。
   - `verify_client_endpoint` 解释为 endpoint tag。
2. **兼容双语义（降风险过渡）**:
   - 若字符串像路径/URL（例如以 `/` 开头或包含 `://`），按旧路径模式走；否则按 tag 查 registry。
   - 缺点: 行为更复杂，但能避免一次性改动 app/bootstrap 的接线。

## 5. 建议的最小验收（L3.2 开工后可直接落地）

- 配置兼容:
  - `verify_client_url` 支持 string 与 object 两种格式解析。
  - `mesh_with` 支持 string 与 object 两种格式解析。
  - `stun` 支持 bool/number/object 解析（与 Go 相同）。
- `/bootstrap-dns` 不依赖 global resolver:
  - 在测试中注入 fake resolver，返回 JSON `{q: ["ip1","ip2"]}`。
- ListenOptions honored（至少 1-2 个可验证项）:
  - `reuse_addr=true` 在预绑定时生效（可通过 socket2/平台 API 断言或至少保证不回退到默认）。
- STUN 默认行为与 Go 一致:
  - 未配置 `stun` 时不启动；
  - 配置启用但缺 listen/listen_port 时，默认 `::` + `3478`。
