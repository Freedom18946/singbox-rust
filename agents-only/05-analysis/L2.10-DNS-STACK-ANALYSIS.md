> **分析日期**: 2026-02-08
> **数据来源**: Go sing-box 1.12.14 源码 + Rust 当前实现 + GUI.for 1.19.0 源码
> **Agent**: Claude Code (Opus 4.6)

## 一、总体判断

Rust DNS 栈已有**大量基础设施**（42+ 文件、完整 trait 层次、10 种 transport、FakeIP、DNS cache、rule engine），但**关键集成链路断裂**——与 L2.8 ConnectionTracker 的情况高度相似。核心问题不是"缺少实现"，而是"已有实现未接线"。

| 维度 | Go | Rust | 差距 |
|------|-----|------|------|
| Transport 种类 | 11 (udp/tcp/tls/https/h3/quic/local/dhcp/hosts/fakeip/tailscale) | 10+ (udp/tcp/dot/doh/doh3/doq/dhcp/local/resolved/tailscale) | hosts/fakeip 未作为 transport |
| DNS Router (wire exchange) | 完整 (rule match → transport → cache → RDRC) | **exchange() 返回 "not yet supported"** | **核心断裂** |
| DNS Client (cache layer) | freelru + 独立缓存 + 去重 + TTL rewrite | V1/V2 cache 存在但**未对接 wire exchange** | 需接线 |
| EDNS0 Client Subnet | 全链路 (config/rule/transport) | UDP 仅有，DoH/DoT/DoQ 缺失 | 需补齐 |
| FakeIP | transport 模式，per-router store，persistent | 全局 env-gated，resolver 短路 | **架构不同** |
| RDRC | storage + 拒绝循环 + 异步持久化 | storage 存在，**未接入解析管线** | 需接线 |
| 反向映射 (reverse mapping) | freelru LRU cache + TTL | 未实现 | 需新增 |
| 独立缓存 (independent_cache) | per-transport 隔离缓存 | config 字段存在，未实现 | 需新增 |
| domain_resolver 链 | server A 用 server B 解析自身域名 | IR 字段存在，未接线 | 需接线 |
| DNS hijack (route action) | hijackDNSStream + hijackDNSPacket | route rule action 类型存在，未实现 | 需新增 |

## 二、Go DNS 架构全景

### 2.1 核心数据流

```
Inbound Traffic
    │
    ▼
[Route Router] ──→ [RuleActionHijackDNS] ──→ [DNS Outbound]
                                                    │
                                                    ▼
                                              [dns.Router]
                                                    │
                                              matchDNS(rules)
                                                    │
                                    ┌───────────────┼───────────────┐
                                    │               │               │
                              DNSRoute          Reject        Predefined
                                    │
                                    ▼
                              [dns.Client]
                                    │
                           ┌────────┼────────┐
                           │                 │
                      cache hit         cache miss
                           │                 │
                           ▼                 ▼
                       return           [RDRC check]
                                            │
                                       ┌────┼────┐
                                       │         │
                                    cached     not cached
                                   rejected       │
                                       │          ▼
                                    return   [Transport.Exchange]
                                    error         │
                                                  ▼
                                         [Response Checker]
                                              │        │
                                           pass     reject
                                              │        │
                                         [cache]  [RDRC save]
                                              │    [try next rule]
                                              ▼
                                         response
```

### 2.2 核心接口

| 接口 | 方法 | 说明 |
|------|------|------|
| `adapter.DNSRouter` | `Exchange(ctx, msg, opts)`, `Lookup(ctx, domain, opts)`, `ClearCache()`, `LookupReverseMapping(ip)`, `ResetNetwork()` | 顶层 DNS 路由 |
| `adapter.DNSTransport` | `Type()`, `Tag()`, `Dependencies()`, `Exchange(ctx, msg)` | wire-format 传输 |
| `adapter.DNSTransportManager` | `Transports()`, `Transport(tag)`, `Default()`, `FakeIP()`, `Create(tag, opts)`, `Remove(tag)` | 传输管理器 |
| `adapter.DNSClient` | `Exchange(ctx, transport, msg, opts, checker)` | 缓存 + RDRC + EDNS0 层 |
| `adapter.FakeIPTransport` | `Store()` (extends DNSTransport) | FakeIP 专用 |
| `adapter.FakeIPStore` | `Contains(addr)`, `Create(domain, isIPv6)`, `Lookup(addr)`, `Reset()` | FakeIP 存储 |
| `adapter.RDRCStore` | `LoadRDRC(transport, qName, qType)`, `SaveRDRC(...)`, `SaveRDRCAsync(...)` | 拒绝缓存 |

### 2.3 DNS 规则系统

**匹配条件** (26+)：inbound, ip_version, query_type, network, auth_user, protocol, domain/suffix/keyword/regex, ip_cidr, ip_is_private, ip_accept_any, source_ip_cidr, source_ip_is_private, source/dest port, process_name/path/regex, package_name, user/user_id, outbound, clash_mode, network_type, wifi_ssid/bssid, rule_set, invert

**动作** (4)：
- `route` — 路由到指定 DNS server，可覆盖 strategy/cache/TTL/client_subnet
- `route-options` — 仅修改选项，不换 server，继续匹配
- `reject` — 返回 REFUSED 或丢弃
- `predefined` — 返回预定义响应

**Address Limit 两阶段匹配**：
1. Phase 1: 匹配请求元数据（domain, source 等）
2. Phase 2: 获得响应后，`MatchAddressLimit()` 检查解析到的 IP
3. 若 Phase 2 失败 → RDRC 记录 → 尝试下一条规则

### 2.4 Transport 详情

| 类型 | 默认端口 | 关键特性 |
|------|---------|---------|
| udp | 53 | 持久连接 + recvLoop + ID 重映射 + truncation → TCP fallback |
| tcp | 53 | length-prefixed wire format |
| tls (DoT) | 853 | 连接池 (linked list) |
| https (DoH) | 443 | HTTP/2 优先 + HTTP/1.1 fallback, POST `application/dns-message` |
| h3 (DoH3) | 443 | QUIC-based HTTP/3 |
| quic (DoQ) | 853 | RFC 9250, 0-RTT, per-query stream |
| local | — | 系统 resolver, /etc/resolv.conf, hosts, search domains, ndots |
| dhcp | 53 | DHCPv4 发现, 1h 刷新 |
| hosts | — | hosts 文件 + predefined entries |
| fakeip | — | 从配置范围分配假 IP |
| tailscale | — | Tailscale MagicDNS |

### 2.5 TransportManager

- 依赖拓扑排序启动
- 默认 transport = 第一个非 FakeIP
- FakeIP 单例（只允许一个）
- 支持热替换 `Create()`/`Remove()`

### 2.6 FakeIP

- **配置**: `inet4_range` (default 198.18.0.0/15), `inet6_range` (default fc00::/18)
- **分配算法**: 双计数器 (inet4Current/inet6Current) + 环绕
- **存储**: Memory (开发) / CacheFile-bbolt (生产)
  - 4 个 bbolt buckets: fakeip_address, fakeip_domain4, fakeip_domain6, fakeip_metadata
  - 异步写入 + 元数据 10s 去抖
- **集成**: 作为 DNS Transport，由规则路由（Exchange 仅处理 TypeA/AAAA）
- **Router 侧**: Lookup() 禁止 FakeIP (需真实 IP)，Exchange() 允许

### 2.7 RDRC

- **存储**: bbolt bucket `rdrc2`, 子桶按 transport 名分
- **Key**: BigEndian(qType) + qName
- **Value**: BigEndian(expiresAt) (Unix timestamp)
- **默认过期**: 7 天
- **流程**: exchange 前查缓存 → 命中返回 ErrResponseRejectedCached; exchange 后响应被拒 → 异步持久化

### 2.8 关键常量

| 常量 | 值 |
|------|-----|
| DefaultDNSTTL | 600s |
| DNSTimeout | 10s |
| FakeIPMetadataSaveInterval | 10s |
| DHCPTTL | 1h |
| RDRC 默认超时 | 7d |

## 五、缺口分析 — 按工作类型分类

### 5.1 核心断裂（必须修复）

| # | 缺口 | 说明 | 复杂度 |
|---|------|------|--------|
| G1 | **DnsRouter.exchange() 未实现** | 整个 wire-format DNS 路由链路不通：rule match → transport exchange → cache → RDRC。当前 `DnsRuleEngine.exchange()` 返回 "not yet supported" | **高** |
| G2 | **RDRC 未接入解析管线** | CacheFileService 有 put_rdrc/get_rdrc，但 DNS resolver/client 从未调用。Go 在 client.Exchange() 中 before/after transport 调用 | **中** |
| G3 | **domain_resolver 链未接线** | DNS server A 用 server B 解析自身域名（如 DoH server 需先解析 dns.google 的 IP）。DnsServerIR 有 address_resolver 字段但未接线 | **中** |

### 5.2 架构差异（需重构/对齐）

| # | 缺口 | 说明 | 复杂度 |
|---|------|------|--------|
| G4 | **FakeIP 非 Transport 模式** | Go: FakeIP 是 DNSTransport，由规则路由选择。Rust: 全局 env-gated，DnsResolver.resolve() 短路。需改为 Transport 模式或在 DnsRouter 层适配 | **中** |
| G5 | **Hosts 非 Transport** | Go: hosts 是 DNSTransport type。Rust: HostsResolver 是独立 Resolver，非 transport 层。GUI 配置 `type: "hosts"` | **低** |

### 5.3 功能缺失（需新增）

| # | 缺口 | 说明 | 复杂度 |
|---|------|------|--------|
| G6 | **Reverse Mapping** | IP → domain 反向映射缓存 (Go: freelru 1024 entries)。用于 connection metadata 显示域名 | **低** |
| G7 | **Independent Cache** | per-transport 隔离缓存 (Go: transportCache)。config 字段存在 | **低** |
| G8 | **DNS Hijack Action** | Route 规则 `hijack-dns` 动作：截获 DNS 流量转交 DNS Router。Go: hijackDNSStream + hijackDNSPacket | **中** |
| G9 | **Address Limit 两阶段匹配** | 规则先匹配请求，获响应后再匹配 IP → 拒绝则尝试下一规则。Go 独有的 DNS 路由特性 | **中** |
| G10 | **route-options 动作** | DNS 规则 `route-options`: 仅修改 strategy/cache/TTL/client_subnet，不换 transport，继续匹配 | **低** |
| G11 | **predefined 动作** | DNS 规则 `predefined`: 返回预定义 rcode/records 响应 | **低** |
| G12 | **disable_expire** | 缓存条目不过期（忽略 TTL）。GUI 有 toggle | **低** |

## 十、实施结果

> **实施日期**: 2026-02-08
> **最终 Parity**: ~99% (208/209)
> **新增测试**: +51 (1492 total)

### 实施方案 vs 分析建议

| 分析建议 | 实际实施 | 差异说明 |
|---------|---------|---------|
| 新增 DnsClient 层 (cache+RDRC+ECS) | 在 DnsRuleEngine 内联实现 | 无需独立 DnsClient 层，rule_engine 本身已是合适的集成点 |
